# 프로젝트 기획서
**장면 위험도 추론 시스템 — Optical Flow + Grad-CAM 기반 Perception 모듈**

---

## 1. 문제 정의

### 기존 방식의 한계

자율주행, 물류 로봇, 서비스 로봇 등 대부분의 로봇 시스템은  
객체 탐지(Object Detection) 모델을 핵심 Perception 모듈로 사용합니다.

YOLO, Faster R-CNN 등의 탐지 모델은 아래 질문에 잘 답합니다.

> **"이 프레임에 무엇이 있는가?"**

그러나 실제 로봇이 안전하게 동작하려면 다음 질문에도 답할 수 있어야 합니다.

> **"지금 이 상황이 위험한가?"**

탐지 모델은 단일 프레임을 기반으로 작동합니다.  
즉, **시간의 흐름** 을 보지 않습니다.  
사람이 탐지됐다는 사실과, 그 사람이 갑자기 도로로 뛰어드는 중이라는 사실은  
단일 프레임으로는 구분이 불가능합니다.

### 내가 주목한 문제

자율주행 사고 사례를 보면, 카메라에 보행자가 분명히 잡혔음에도  
차량이 제때 반응하지 못한 경우가 있습니다.

"탐지는 됐는데, 왜 멈추지 않았을까?"

이 질문에서 이 프로젝트가 출발했습니다.  
문제는 탐지 자체가 아니라, **탐지 결과를 해석하는 레이어의 부재**입니다.

로봇이 "사람이 있다"는 것을 아는 것과  
"이 사람의 움직임이 위험 신호인가"를 판단하는 것은 전혀 다른 능력입니다.  
이 프로젝트는 후자를 직접 설계해보는 실험입니다.

---

## 2. 접근 방식

### 핵심 아이디어

단일 프레임이 아닌 **연속 프레임 간의 움직임 변화(Optical Flow)** 를 입력으로 사용합니다.  
움직임 벡터의 크기와 방향 패턴이 위험 상황에서 다르게 나타난다는 가설을 검증합니다.

### 파이프라인 설계 근거

| 단계 | 선택한 방법 | 이유 |
|---|---|---|
| 움직임 추출 | Optical Flow (Farneback) | 전체 픽셀의 흐름을 밀집하게 추출, 패턴 분석에 적합 |
| 특성 표현 | Magnitude Map | 벡터 크기를 이미지로 변환 → CNN 입력으로 활용 가능 |
| 분류 | ResNet + Fine-tuning | 소량 데이터에서도 강력한 특성 추출기 활용 |
| 해석 | Grad-CAM | 판단 근거를 공간적으로 시각화, 오류 디버깅 가능 |

### 기존 프로젝트와의 차이

- 일반적인 이상 행동 탐지 프로젝트: 영상 → 탐지 → 분류 결과 출력
- **이 프로젝트**: 영상 → 움직임 분석 → 분류 → **왜 위험하다고 판단했는가 시각화**

결과의 정확도보다, **모델의 판단 과정을 해석할 수 있는 구조** 를 목표로 합니다.

## 3. 로봇 분야와의 연결

이 프로젝트가 다루는 문제는 실제 로봇 시스템에서 다음과 같이 연결됩니다.

- **자율주행**: 보행자 행동 예측, 긴급 제동 판단
- **물류 로봇**: 사람과의 충돌 예측, 경로 회피 트리거
- **서비스 로봇**: 이상 상황 감지, 안전 알림

궁극적으로 이 모듈은 로봇의 **Safety Layer** — 상위 판단 명령을 하위에서 검토하고  
차단할 수 있는 안전망 — 의 프로토타입으로 발전할 수 있습니다.

## 4. 예상 한계 (시작 전 솔직한 전망)

| 한계 | 이유 |
|---|---|
| 야간·저조도 영상에서 성능 저하 예상 | Optical Flow는 밝기 일정성 가정에 의존 |
| 카메라 자체가 흔들릴 때 오탐 가능 | 배경 움직임과 객체 움직임 구분 어려움 |
| 이진 분류(위험/정상)의 한계 | 위험도는 연속적 스펙트럼, 이분법은 단순화 |
| 실시간 처리 미검증 | 이 프로젝트는 오프라인 분석에 집중 |

> 이 한계들은 프로젝트 완료 후 실제로 관찰됐는지, 어떻게 나타났는지를  
> 결과 분석 섹션에 업데이트할 예정입니다.
