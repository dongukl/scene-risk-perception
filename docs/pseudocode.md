# 수도 코드 및 설계
**장면 위험도 추론 시스템 — 실제 구현 기반 설계 문서**

> 이 문서는 코드 작성 전 설계한 내용을 바탕으로,
> 실제 구현 과정에서 발견한 문제와 수정 사항을 함께 기록한 문서입니다.

---

## 전체 흐름

```
[입력] UCSD Ped2 영상 프레임 (.tif)
    ↓
[1단계] 프레임 로드 및 전처리
    ↓
[2단계] Optical Flow 계산 → Magnitude Map 생성
    ↓
[3단계] 데이터셋 구성 (훈련 70% / 검증 15% / 테스트 15%)
    ↓
[4단계] ResNet18 전이학습 (layer4 + conv1 + fc)
    ↓
[5단계] Grad-CAM 시각화
    ↓
[출력] 분류 결과 + 판단 근거 히트맵 + 분석 리포트
```

---

## 1단계: 프레임 로드 및 전처리

```
함수: load_frames(폴더 경로)

    입력: .tif 프레임이 담긴 폴더 경로
    출력: 프레임 이미지 리스트

    과정:
        폴더 안의 .tif 파일을 시간 순서로 정렬
        반복 (각 파일마다):
            PIL로 이미지 읽기  ← cv2.imread가 .tif를 못 읽는 문제 발견 → PIL로 해결
            그레이스케일로 변환 (UCSD Ped2는 이미 그레이스케일)
            224x224로 크기 통일 (ResNet 입력 크기)
            리스트에 추가
        반환: 프레임 리스트
```

**실제 구현에서 발견한 문제**
- `cv2.imread`가 `.tif` 확장자를 읽지 못하는 문제 발생
- `PIL.Image.open()`으로 교체하여 해결

## 2단계: Optical Flow 계산

```
함수: compute_optical_flow(프레임A, 프레임B)

    입력: 연속된 두 프레임 (그레이스케일, 224x224)
    출력: Magnitude Map (224x224) - 움직임 크기 이미지

    과정:
        Farneback 알고리즘으로 두 프레임 간 흐름 벡터 계산
            pyr_scale=0.5, levels=3, winsize=15
            iterations=3, poly_n=5, poly_sigma=1.2
        흐름 벡터를 가로(dx), 세로(dy) 성분으로 분리
        magnitude = sqrt(dx² + dy²)
        0~255 범위로 정규화
        반환: magnitude 이미지


함수: build_dataset(훈련 경로, 테스트 경로)

    과정:
        Train 폴더 → 연속 프레임 쌍마다 Flow 계산 → 레이블 0 (정상)
        Test 폴더  → _gt 폴더 존재하면 레이블 1 (이상), 없으면 0 (정상)
            ※ _gt 폴더 = Ground Truth, 이상 프레임 정답 레이블

    결과:
        전체 13,930개 샘플
        정상(0): 11,940개 (86%)
        이상(1):  1,990개 (14%)
```

## 3단계: 데이터셋 구성

```
분할 전략:
    전체 → 훈련 70% / 검증 15% / 테스트 15%
    stratify=labels 옵션으로 정상/이상 비율 유지

결과:
    훈련셋:  약 8,900개 → DataLoader (배치 32, shuffle=True)
    검증셋:  약 2,200개 → DataLoader (배치 32, shuffle=False)
    테스트셋: 약 2,800개 → DataLoader (배치 32, shuffle=False)

입력 형태 변환:
    (N, 224, 224) → (N, 1, 224, 224)  ← 채널 차원 추가 (ResNet 입력 형태)
    픽셀값 0~255 → 0~1로 정규화
```

## 4단계: ResNet18 전이학습

```
모델 설계:
    사전학습된 ResNet18 (ImageNet) 불러오기
    
    수정 사항:
        conv1: 3채널 → 1채널 (Optical Flow는 그레이스케일)
        fc:    1000클래스 → 2클래스 (정상 / 이상)
    
    학습 전략:
        전체 파라미터 고정 (freeze)
        학습 허용: conv1 + layer4 + fc
        → 전체의 약 30% 파라미터만 학습
        
        ※ 처음엔 conv1 + fc만 학습 (0.5%) → 성능 부족
        ※ layer4 추가 학습으로 성능 대폭 향상


손실 함수:
    CrossEntropyLoss (가중치 적용)
    
    클래스 불균형 문제 발견:
        정상 86% vs 이상 14% → 모델이 전부 정상으로만 예측
        해결: 클래스 가중치 부여
            정상 가중치  = 전체 / (2 × 정상 수) ≈ 0.58
            이상 가중치 = 전체 / (2 × 이상 수) ≈ 3.50


학습 설정:
    옵티마이저: Adam (lr=1e-4)
    스케줄러:   StepLR (step=5, gamma=0.5)
    에폭:       20


평가 기준 우선순위:
    1순위: Recall  → 위험을 놓치지 않는 것
    2순위: Precision → 오탐율 관리
    3순위: F1 Score → 균형 지표
```

**시도별 결과 기록**

| 시도 | 변경 내용 | Recall | Precision | 문제 |
|---|---|---|---|---|
| 1차 | conv1+fc, 가중치 3.5 | 0.87 | 0.23 | 오탐율 너무 높음 |
| 2차 | 가중치 2.0으로 낮춤 | 0.28 | 0.33 | Recall 급락 |
| 3차 | layer4 추가, 가중치 3.5 | **0.937** | **0.944** | 안정적 |

## 5단계: Grad-CAM 시각화 (진행 예정)

```
함수: apply_gradcam(모델, 이미지)

    입력: 학습된 모델, Magnitude Map 이미지 1장
    출력: Grad-CAM 히트맵 (원본 위에 오버레이)

    과정:
        이미지를 모델에 입력 → 예측 클래스 확인
        마지막 Conv 레이어(layer4)에서 클래스별 기울기 계산
        기울기의 전역 평균 풀링(GAP) → 레이어별 가중치 산출
        가중치와 특성맵의 가중합 → ReLU 적용
        원본 크기로 업샘플링 → 원본 이미지에 오버레이


함수: analyze_failures(모델, 테스트셋)

    과정:
        False Negative 수집: 실제 위험인데 정상으로 분류
        False Positive 수집: 실제 정상인데 위험으로 분류
        각 케이스에 Grad-CAM 적용
        패턴 분석 및 리포트 저장
```
